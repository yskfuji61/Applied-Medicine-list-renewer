Requirements Specification

薬剤リスト変換アプリ 要件定義書

旧採用医薬品リストのテーブル形式を、260508_Musashino_採用医薬品配下のCSV群から再現・生成するためのローカル実行型アプリケーションの要件定義。 GPT-5.4 を VS Code 内の Copilot Chat で活用しつつ、本番変換処理は決定論的なロジックとテストで品質保証することを前提とする。

作成日 2026-05-12
対象リポジトリ 薬剤リスト
開発前提 VS Code + Copilot Chat + GPT-5.4
品質方針 再現性 / 監査性 / 保守性

1. 背景と目的

現在は新採用医薬品CSV群を都度GPTに渡して整形しているが、この運用は再現性、監査性、処理時間、改版対応、コストの面で不安定である。 本アプリの目的は、入力CSV群をローカルで読み込み、旧採用医薬品リストと同等の表形式ファイル群を半自動または自動で生成できるようにすることである。

達成すべき状態

  • 新採用CSV群から旧採用形式の出力を安定して再生成できる。
  • 変換結果の根拠がコード、設定、ログで追跡できる。
  • 曖昧な一致は人が確認でき、確認結果を再利用できる。
  • 将来のCSVレイアウト変更に対して設定更新で追従しやすい。

採らない方針

  • 本番変換を毎回LLMの自由生成に依存しない。
  • 単純なCSV結合だけで帳票を作らない。
  • 未確定一致を自動確定しない。
  • 外部APIへ実データを恒常的に送信しない。

3. スコープ

対象範囲

  • 複数CSVの読み込み、文字コード判定、列マッピング、正規化。
  • 標準スキーマへの統合。
  • 採用判定、重複解消、未確定候補抽出。
  • 旧採用形式に対応する派生CSVの出力。
  • 差分レポート、実行ログ、ルール管理。
  • ローカルGUIまたはCLIによる運用。

対象外

  • 院内システムとの直接連携。
  • 複数病院向けの一般化プロダクト化。
  • クラウド常駐サービス化。
  • OCRによる紙帳票読み取り。

4. 想定利用者

利用者 主目的 必要な操作

メンテナ ルール更新と品質管理 例外辞書編集、列マッピング修正、テスト実行
監査担当 変換根拠を追跡する 差分確認、実行履歴確認、ログ閲覧

5. 目標システム像

アプリは「決定論的な変換コア」を中心に構成し、その外側に CLI と軽量GUIを置く。GPT-5.4 は開発補助と例外候補整理に利用し、日常運用の主処理はローカルロジックで完結させる。

論理構成

  • 入力層: CSV検出、文字コード推定、必須列検査、入力プロファイル作成。
  • 正規化層: 列名統一、文字種統一、名称正規化、欠損補完、剤形区分付与。
  • 照合層: 点数コード、YJコード、名称、例外辞書による多段一致。
  • 判定層: 採用可否、保留、重複解消、優先ソース選択。
  • 出力層: 標準マスタ、旧採用互換CSV群、差分レポート、監査ログ。
  • 運用層: CLI、GUI、設定ファイル、テスト、パッケージング。

6. 機能要件

6.1 入力処理

  • 利用者は入力対象フォルダまたはCSVファイルを選択できること。
  • アプリは対象CSVの存在、読込可否、文字コード、列名一覧、件数を検査できること。
  • 文字コードは UTF-8、UTF-8 BOM、CP932、Shift_JIS を優先候補として再試行できること。
  • 大容量CSVはストリームまたは遅延評価で処理できること。

6.2 標準化

  • 入力ごとに異なる列名を標準スキーマへマッピングできること。
  • 製品名、一般名、YJコード、点数コード、薬効コードなど主要列を標準化できること。
  • 名称正規化は全角半角、ダッシュ種別、空白種別、括弧注記、記号ゆれを吸収できること。
  • 各レコードに由来ファイル名、元行番号、変換時刻を保持できること。

6.3 照合・判定

  • 一致戦略は点数コード完全一致、YJコード一致、正規化名称一致、例外辞書一致、曖昧一致候補提示の順に適用できること。
  • 曖昧一致は自動確定せず、候補とスコアを提示すること。
  • 重複候補が出た場合は優先規則に従い1件へ統合、または保留にできること。
  • 採用対象外、情報不足、列欠損は理由つきで除外または保留にできること。

6.4 出力

  • 標準化済みマスタCSVを出力できること。
  • 旧採用医薬品リスト互換の派生CSV群を出力できること。
  • 少なくとも作業シート、一般名順、製品名順、薬効順、薬効コード順に相当する出力を生成できること。
  • 出力ごとに件数、生成時刻、使用ルールバージョンを付記または別ファイルに保存できること。

6.5 差分・監査

  • 前回出力との件数差分、新規、削除、名称変更、コード変更、保留件数を比較できること。
  • 実行ログに入力ハッシュ、使用設定、変換件数、警告件数、エラー件数を記録できること。
  • 監査用に「なぜその行が採用または除外されたか」を追跡できること。

8. ルール管理要件

設定ファイル化すべきもの

  • 入力ファイルごとの列マッピング定義
  • 採用対象抽出ルール
  • 除外ルール
  • 名称置換辞書
  • 例外的一致辞書
  • 出力ビューごとの並び順定義

更新原則

  • ルール変更はコード変更と切り分ける。
  • ルール変更ごとにバージョンと変更理由を持つ。
  • 本番運用で確定した手修正は辞書へ還元する。
  • 辞書変更時はゴールデンテストを再実行する。

9. 非機能要件

分類 要件
性能 数千行から数万行規模のCSVを実用時間で処理できること。大容量CSVでもメモリ逼迫を避けること。
再現性 同じ入力、同じ設定、同じアプリ版数なら常に同じ出力になること。
監査性 入力、変換、出力、判定理由、ルール版数が追跡可能であること。
保守性 入力フォーマット変更時に設定ファイルと限定的なコード修正で追従できること。
可観測性 警告件数、保留件数、未一致件数、処理時間を可視化できること。
可用性 ネットワーク非依存のローカル実行が可能であること。
セキュリティ 実データを無断で外部送信しないこと。必要時も明示操作がない限りAI送信を行わないこと。

10. 推奨技術構成

必須候補

  • 言語: Python 3.12
  • 表データ処理: Polars
  • 補助クエリ: DuckDB
  • CLI: Typer
  • 検証: Pydantic v2 / Pandera
  • 曖昧一致: RapidFuzz
  • テスト: pytest / Hypothesis

GUI候補

  • 初期版: Streamlit または Textual
  • 配布重視: Tauri フロントエンドを後段で検討
  • 初期段階ではCLI優先でコアを安定化する

技術原則

  • 変換コアは UI 非依存のライブラリとして設計する。
  • AI利用箇所は補助的に限定し、変換結果の決定はコードと設定で行う。
  • CSV処理の境界で型検証を行い、不正値は即時に警告または失敗させる。

11. GPT-5.4 と Copilot Chat の利用方針

用途 利用可否 理由
要件整理、列マッピング案、テストケース案 推奨 探索とドラフト生成に強い。
例外辞書の候補作成 推奨 未一致行の整理を高速化できる。
本番変換の最終判断 非推奨 再現性、監査性、誤判定統制の観点で不適。
実データの常時外部送信 禁止 セキュリティと運用安定性の観点で不適。

Copilot Chat への依頼は大粒度にせず、読み込み、正規化、照合、出力、テストの単位で分割する。これによりレビューと差分管理がしやすくなる。

12. テスト要件

  • 文字コード判定テスト: UTF-8、BOM付きUTF-8、CP932 の読込確認。
  • 列マッピングテスト: 必須列欠落時の失敗確認、既知フォーマットの正常読込確認。
  • 名称正規化テスト: ダッシュ、空白、全半角、括弧付き注記の冪等性確認。
  • 照合テスト: 点数コード一致、YJコード一致、辞書一致、保留判定の確認。
  • ゴールデンファイルテスト: 既知入力から既知出力が完全一致すること。
  • 回帰テスト: ルール更新やライブラリアップデート後も主要出力が崩れないこと。

13. 受け入れ条件

  • 入力CSV群から標準化マスタを生成できること。
  • 旧採用互換の複数CSVを規定の並び順で出力できること。
  • 未確定候補が一覧化され、手動確認後に再利用できること。
  • 前回差分レポートを生成できること。
  • 同一入力で出力が安定すること。
  • テストスイートが整備され、主要経路で自動検証できること。

14. 実装フェーズ案

15. リスクと対策

リスク 影響 対策
文字コード判定失敗 文字化け、照合率低下 候補順で再試行し、失敗時はプレビューで確認させる。
列レイアウト変更 読込失敗 列マッピング設定を外出しし、必須列検査で早期検知する。
名称揺れ増大 未一致増加 正規化ルールと例外辞書を継続拡張する。
LLM依存の拡大 再現性低下 AIは補助用途に限定し、本番変換から切り離す。
本書は、現時点で確認できたCSV構造と運用前提に基づく実装用要件定義書である。次工程では列マッピング仕様書、例外辞書仕様、CLI入出力仕様へ分解して実装可能な単位へ落とし込む。
