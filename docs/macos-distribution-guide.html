macOS App / Distribution

薬剤リスト変換アプリ 配布ガイド

この文書は、Python 同梱の完全スタンドアロン .app ビルド、社内配布、Developer ID 署名、notarization、公開前チェックまでを 1 枚で確認できる運用ガイドです。標準配布物は Python 未導入の macOS 端末でもそのまま実行できます。

生成される成果物

.app バンドル

dist/macos-standalone-release/薬剤リスト変換アプリ.app

Finder からダブルクリックで実行可能。PyInstaller により Python ランタイムを同梱。

配布フォルダ

dist/macos-standalone-release

app 本体、config、docs、入力データ、参照データ、README、リリースノート雛形をまとめた実行用フォルダ。

配布 ZIP

dist/薬剤リスト変換アプリ-standalone-macos.zip

公開配布対象。署名と notarization はこの ZIP をベースに実行する。

ビルド手順

  1. プロジェクトルートで次を実行する。
./scripts/build_macos_standalone.sh
  1. 完了後に dist/macos-standalone-release と dist/薬剤リスト変換アプリ-standalone-macos.zip を確認する。
  2. app 実行時は docs/audit-reports/latest/diff-report.html が自動で開く。

現在のランタイム前提

OSmacOS 12 以上を想定
Python不要。PyInstaller で同梱済み。
入力260508_Musashino_採用医薬品/references を配布フォルダ内に同梱。
参照旧採用医薬品リスト を配布フォルダ内に同梱。
設定config/defaults.json を配布フォルダに同梱し、app から参照。
レポート配布フォルダ内の docs/audit-reports/latest に出力される。

社外公開前には Developer ID 署名と notarization を必須にしてください。未署名配布では Gatekeeper 警告が強く出ます。

社内配布チェックリスト

  1. 最新コードで PYTHONPATH=src python3 -m unittest tests.test_profile を通す。
  2. ./scripts/build_macos_standalone.sh を実行する。
  3. 生成された .app を 1 回実行し、HTML レポートが自動で開くことを確認する。
  4. 配布 ZIP を別ディレクトリに展開し、そこから再実行して相対パス依存がないことを確認する。
  5. 配布する ZIP にバージョン、作成日、差分要約を添える。
  6. 配布フォルダに同梱された README.md と RELEASE_NOTES_TEMPLATE.md を更新する。

公開前に必須の作業

  1. 個人名、ローカル絶対パス、社内限定ファイル名がレポートや docs に残っていないか確認する。
  2. 入力 CSV や旧採用リストに第三者提供不可の情報が含まれていないか確認する。
  3. 配布物の README を追加し、前提 OS、署名状態、既知制約を明記する。
  4. リリースごとの内容は RELEASE_NOTES_TEMPLATE.md から起票して確定版を残す。
  5. アプリ名、バージョン、ビルド日、変更履歴を固定する。
  6. 初回起動時の Gatekeeper 警告への案内文を準備する。

署名と公証の前提

証明書Keychain に Developer ID Application 証明書が入っていること。
notarytoolxcrun notarytool store-credentials 済みの keychain profile を持つこと。
環境変数APPLE_SIGN_IDENTITY と APPLE_NOTARY_PROFILE を設定すること。
export APPLE_SIGN_IDENTITY="Developer ID Application: YOUR NAME"
export APPLE_NOTARY_PROFILE="your-notary-profile"
./scripts/sign_and_notarize_macos.sh

このスクリプトは codesign、ZIP 再生成、notarytool submit、stapler、spctl 検証、SHA256 出力までを一括で行います。

GitHub Actions 公開パイプライン

.github/workflows/macos-standalone-release.yml を追加済みです。workflow_dispatch でテスト、standalone build、証明書 import、署名、公証、artifact upload まで流せます。

必須 secretsMACOS_CERTIFICATE_BASE64, MACOS_CERTIFICATE_PASSWORD, KEYCHAIN_PASSWORD, APPLE_SIGN_IDENTITY, APPLE_ID, APPLE_APP_SPECIFIC_PASSWORD, APPLE_TEAM_ID
notary profileworkflow 内で ci-notary として一時 keychain に保存
成果物署名済み ZIP と release フォルダを artifact として保存
secret 自動投入./scripts/configure_github_actions_secrets.sh を利用可能。ただし gh auth login と GitHub remote が前提。

Trigger: workflow_dispatch
Main steps:
1. setup-python
2. unittest
3. build_macos_standalone.sh

5. sign_and_notarize_macos.sh

公開パイプライン

  1. テスト完了
  2. ./scripts/build_macos_standalone.sh でスタンドアロン版を生成
  3. ./scripts/sign_and_notarize_macos.sh で署名・公証
  4. spctl -a -vvv でアプリ受理を確認
  5. ZIP と .sha256 を公開
  6. README、変更履歴、既知制約を添付

推奨の公開フロー

  5. 署名・公証
